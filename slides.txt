SLIDE 1 — TITLE
LOOM: Surfacing Latent Structure Across LLM Conversations for Transparent Personalization


SLIDE 2 — PROBLEM
- People use LLM chatbots daily for knowledge-seeking, yet each conversation usually starts from zero
- Over weeks/months, users accumulate hundreds of scattered chats that contain a latent structure — but it remains invisible to both the user and the AI
- Loom surfaces this latent structure and embeds it directly into the chat workflow — making personalization transparent, editable, and cumulative

SLIDE 3 — LATENT STRUCTURE
- This latent structure has multiple dimensions:
  - Topical clusters: conversations naturally group around recurring themes
  - Hierarchical user status: within each topic, there is a general layer (background, goals, proficiency) and a specific layer (individual concepts explored, each with an inferred understanding level)
  - Cross-session continuity: new conversations don't just relate to past ones — they can build on them, turning isolated Q&A into cumulative understanding
  - Unexplored adjacencies: each topic sits within a broader space of related areas the user hasn't encountered yet


SLIDE 4 — LOOM'S APPROACH
Thesis: We can surface this latent structure and embed it directly into the chat workflow — making personalization transparent, editable, and useful — without requiring a separate knowledge management tool.

Loom infers topical structure from everyday chats and provides three modules in a sidebar, each making a different dimension of latent structure visible:

Module 1 — Status Summary ("What does the AI think it knows about me?")
- Externalizes the AI's inferred user model as a structured, editable representation per topic
- Two levels: overview (background, goals, proficiency) and specifics (individual concepts with inferred understanding levels)
- User can inspect and edit — mixed-initiative control over personalization
- Feeds into LLM responses as injected context and improves Module 2's retrieval quality

Module 2 — Cross-Session Connections ("How does this relate to what I've explored before?")
- Surfaces relationships between the current conversation and past chats via semantic similarity
- LLM annotates its response with inline markers where genuine connections exist
- Interactive popovers: explains the relationship, offers "Build on this" (injects past context) and "Go to chat" (navigates to linked conversation)
- Connection cards in sidebar with cross-highlighting on hover

Module 3 — New Directions ("What should I ask next that I don't know to ask?")
- Uses user status, covered concepts, and current conversation to suggest 2–3 adjacent unexplored areas
- Draggable/clickable into chat input; also appear as suggestion cards on the welcome screen as re-entry points


SLIDE 5 — STUDY PLAN
- Participants: 10–12 university students, within-subject design
- Conditions: Loom vs. baseline (LLM chat with memory-as-a-list), each used for 5 days, counterbalanced order
- Minimum usage: ~5 chats / 10 messages per day
- Logged data:
  - Queries, topics, interactions with each module
  - Summary edits, topic creation/removal (mixed-initiative signals)
  - Connection clicks, "Build on this" usage, new direction adoption
- Post-study: 30-min qualitative interview
  - Personalization accuracy and perceived quality
  - Response helpfulness vs. baseline
  - Controllability and trust in the user model
  - Whether surfaced structure changed how they used the chatbot


SLIDE 6 — OPEN QUESTIONS
- Framing: "Latent structure in everyday LLM use" — broad enough to cover all four dimensions (topics, status, connections, directions) under one concept
- Contribution positioning: Personalization transparency? Conversational knowledge structuring? Both?
- Technical novelty: The integration of embedding-based retrieval, structured user modeling, and proactive suggestion within a single chat interface
- Study design: Right dosage of minimum usage? Best measures for whether structure surfacing changes behavior vs. just awareness?
